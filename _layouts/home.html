   Vulnerabilities are a known problem in today’s software supply chain (SSC). Because software components often depend on external dependencies (libraries), attackers can exploit vulnerabilities in one component to spread malicious code, which creates security risks for projects that rely on those vulnerable components. Existing research addresses these challenges through software composition analysis (SCA). Beyond the revelation, SCA tools employ various vulnerability detection and remediation approaches, resulting in different outcomes. However, ineffective solutions may introduce new issues, such as compilation failures, dependency conflicts, and additional vulnerabilities. To address this, we propose Vuln Scan and Pro (VULNSCANPRO), a robust solution for detection and remediation vulnerabilities in Maven projects. Specifically, VULNSCANPRO builds a fine-grained method graph to identify bloated code and vulnerable dependencies. The method graph is mapped to the project’s dependency tree, forming a comprehensive dependency graph that highlights vulnerabilities. Based on this analysis, we propose three solutions for vulnerability remediation: (1) Removing bloated dependencies that contain vulnerabilities, thereby resolving security risks and reducing maintenance overhead. (2) Upgrading vulnerable dependencies to the closest non-vulnerable versions, while pinning the versions of transitive dependencies introduced by the vulnerable dependency, in order to mitigate compatibility issues and prevent the introduction of new vulnerabilities. (3) Eliminating bloated code within vulnerable dependencies, particularly when security patches are either incompatible or absent. Experimental results show that these solutions effectively mitigate vulnerabilities and enhance the overall security of the project.
